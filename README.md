# Telegram 自动化下载工具

一个基于 .NET 6.0 和 Selenium 的高性能 Telegram 频道内容自动化下载工具。本工具支持多线程并发下载、断点续传、智能重试等特性,并提供了完善的日志记录和异常处理机制。

## 核心特性

### 自动化能力
- 🤖 智能模拟用户行为
  - 仿真键盘输入
  - 动态延迟控制
  - 自动状态检测
- 🚀 高效并发下载
  - 多线程下载支持
  - 智能任务调度
  - 断点续传功能
- 🔄 可靠性保障
  - 智能重试机制
  - 异常自动恢复
  - 会话状态维护

### 安全性
- 🔒 安全的浏览器配置
  - 沙箱隔离
  - 安全策略控制
  - 用户代理伪装
- 🛡️ 资源保护
  - 自动资源释放
  - 内存使用优化
  - 并发数量控制

### 用户体验
- 📝 完善的日志系统
  - 分级日志记录
  - 自动日志归档
  - 详细错误追踪
- 🎯 友好的用户界面
  - MVVM 架构设计
  - 实时进度反馈
  - 清晰的状态展示

## 系统要求

### 运行环境
- 操作系统: Windows 10/11
- 运行时: .NET 6.0 或更高版本
- 浏览器: Chrome 122.0.6261.5700 或更高版本

### 硬件要求
- CPU: 双核处理器或更高
- 内存: 4GB RAM 或更高
- 存储: 500MB 可用空间

## 技术架构

### 核心框架
- **.NET 6.0**: 提供高性能的运行时支持
- **WPF**: 实现现代化的用户界面
- **Selenium 4.18.1**: 提供强大的浏览器自动化能力

### 关键组件
- **WindowsInput 6.4.1**: 实现精确的键盘输入模拟
- **NLog 5.2.8**: 提供企业级日志管理
- **ChromeDriver**: 实现浏览器精确控制

### 设计模式
- **MVVM**: 实现界面与逻辑分离
- **命令模式**: 处理用户交互
- **工厂模式**: 管理对象创建
- **观察者模式**: 处理状态更新
- **依赖注入**: 实现模块解耦

## 项目结构

```
TelegramAutomation/
├── Commands/                    # 命令模式实现
│   └── RelayCommand.cs         # 通用命令实现类
│
├── Models/                      # 数据模型
│   ├── AppSettings.cs          # 应用程序配置
│   └── DownloadConfiguration.cs # 下载配置
│
├── Services/                    # 业务服务
│   ├── DownloadManager.cs      # 下载管理器
│   └── MessageProcessor.cs      # 消息处理器
│
├── Themes/                      # 主题资源
│   └── Default.xaml            # 默认主题样式
│
├── ViewModels/                  # MVVM 视图模型
│   ├── MainViewModel.cs        # 主窗口视图模型
│   └── ViewModelBase.cs        # 视图模型基类
│
├── Views/                       # 视图层
│   ├── MainWindow.xaml         # 主窗口界面
│   └── MainWindow.xaml.cs      # 主窗口代码
│
├── AutomationController.cs      # 自动化控制器
├── App.xaml                     # 应用程序定义
├── App.xaml.cs                 # 应用程序入口
├── nlog.config                 # NLog 配置文件
├── appsettings.json           # 应用程序配置文件
└── TelegramAutomation.csproj  # 项目文件
```

## 模块说明

### 1. 核心模块 (Core)
- **AutomationController**: 自动化控制核心，管理浏览器操作和登录流程
- **Models**: 定义数据结构和配置模型
- **Services**: 提供下载和消息处理服务

### 2. 用户界面 (UI)
- **Views**: WPF 界面定义
  - MainWindow.xaml: 主窗口界面
  - MainWindow.xaml.cs: 主窗口代码
- **ViewModels**: MVVM 模式的视图模型实现
- **Commands**: 命令模式实现，处理用户操作

### 3. 配置管理
- **AppSettings**: 应用程序配置管理
- **DownloadConfiguration**: 下载相关配置
- **nlog.config**: 日志配置
- **appsettings.json**: 应用程序配置文件
